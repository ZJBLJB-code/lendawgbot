# Len Dawg dashboard

A read-only public dashboard for one stakeholder. Shows whether an
automated /MES futures bot is making or losing money, day-by-day, trade-by-trade.

Live: https://zjbljb-code.github.io/lendawgbot/

## What the page shows (4 sections)

1. **Header** — bot name, instrument, freshness dot.
2. **Hero P&L** — account change since launch (the one number).
3. **Daily P&L** — every trading day with that day's net.
4. **All trades** — every closed trade, filterable by date range, side, outcome, strategy.

That's it. No personas, no commentary, no narrative.

## Hard rule

Every dollar number on the page traces to a real journal event. The
authoritative source for "is the account up or down" is the
`equity_after_usd` field on `position_closed` events:

```
starting_equity = first_close.equity_after_usd − first_close.net_pnl_usd
current_equity  = last_close.equity_after_usd
account_change  = current_equity − starting_equity
```

The sum of per-trade `net_pnl_usd` is a **secondary** number. When it
diverges from `account_change` (it always does, by broker fees and
slippage), the dashboard surfaces the gap explicitly under the hero —
never hidden, never silently reconciled.

If a value can't be computed from a real event, the UI renders an em-dash.
There are no synthetic defaults. `tests/test_no_synthetic_defaults.py`
greps for the prior-art lies (`STARTING_EQUITY_DEFAULT`, `|| 10000`,
`"$10K"`, persona/skeptic/brag copy) and fails the build red on any hit.

## Architecture

```
quant-bot journal/live/*.jsonl
        |
        v
tools/publisher.py  --reads-->  data/dashboard.json   (committed)
        |
        v
scripts/build.py    --inlines->  dist/index.html      (deployed by CI)
```

The publisher runs **locally** on the operator's machine — that is the
only place with access to the unsanitized journal. CI in this repo only
reads the committed `data/dashboard.json`, runs tests + the PII grep,
inlines the JSON into the template, and deploys to GitHub Pages.

## Local run

```bash
# 1. Recompute data/dashboard.json from your local quant-bot journal
python3 tools/publisher.py /path/to/quant-bot

# 2. Verify nothing diverges from raw events
python3 scripts/verify_against_truth.py --qb-root /path/to/quant-bot

# 3. Build dist/index.html and open it
python3 scripts/build.py
open dist/index.html

# 4. Commit + push — GitHub Pages redeploys automatically
git add data/dashboard.json && git commit -m "refresh: dashboard" && git push
```

## Tests

```bash
pip install pytest
python3 -m pytest tests/ -v
```

- `test_no_synthetic_defaults.py` — greps source for placeholder copy + numeric fallbacks.
- `test_real_bot_correctness.py` — runs publisher against pinned fixtures, asserts every headline number.

## Source layout

```
tools/publisher.py               # journal -> data/dashboard.json
scripts/build.py                 # data/dashboard.json + template -> dist/index.html
scripts/verify_against_truth.py  # independent re-check of headline numbers
src/template.html                # the entire UI (single file, no framework)
data/dashboard.json              # committed canonical, regenerated by publisher
.github/workflows/pages.yml      # CI: test + grep + build + deploy
tests/                           # pytest
```

Total source code is intentionally under 1,500 lines.

## Privacy

The committed `data/dashboard.json` exposes account change in dollars,
absolute starting/current equity, and per-trade P&L (the whole point of
the dashboard). It does NOT expose: account ID, IBKR client ID,
host/PID, drawdown %, risk-budget knobs, or any of the raw event field
names (`equity_usd`, `equity_after_usd`, `net_pnl_usd`, etc.) that the
PII grep blocks.

Not financial advice.
